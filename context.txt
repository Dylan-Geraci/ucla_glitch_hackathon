################################################################################
SECTION 1 — PROJECT IDENTITY & CONSTRAINTS
################################################################################

PROJECT NAME: Project Specter

MISSION:
  An Electron-based AI Sidekick that silently observes a Selenium-controlled
  Chromium browser. It listens to the user's microphone, watches the screen,
  and provides real-time verbal and visual assistance — highlighting answers
  directly in the browser DOM and speaking explanations aloud — all without
  a traditional backend server.

NON-NEGOTIABLE ARCHITECTURAL RULES:
  [RULE-01] NO backend server. Zero. No FastAPI, no Express, no localhost server.
            All orchestration logic runs in the Electron Main process (Node.js).

  [RULE-02] The "frontend" IS the Selenium-controlled browser.
            There is no custom React/Next.js app for the user to interact with.
            The browser Selenium controls IS what the user sees and uses.

  [RULE-03] The only custom Electron UI is the HUD Overlay BrowserWindow —
            a small, transparent, always-on-top status bubble. Nothing more.

  [RULE-04] Gemini Multimodal Live API is the ONLY AI model used at MVP.
            Do not add fallback models, secondary APIs, or local models.

  [RULE-05] IPC channel names are FROZEN. Use exactly the names in Section 4.
            Do not rename, abbreviate, or add new channels without updating this file.

  [RULE-06] Selenium function names exposed as Gemini tools are FROZEN.
            Use exactly the names in Section 5.

FULL DATA FLOW — AUDIO INPUT PATH:
  1. HUD Renderer captures mic → PCM Int16 @ 16kHz chunks
  2. HUD Renderer → ipcRenderer.send('gemini:audio-chunk', { data, mimeType })
  3. Main receives → forwards to Gemini WebSocket as realtimeInput.audio
  4. Gemini VAD detects end-of-speech → processes multimodal context
  5a. If conversational: Gemini emits serverContent with audio parts
      → Main receives PCM 24kHz → ipcMain sends 'hud:audio-out' to Renderer
      → Renderer plays via AudioContext
  5b. If action required: Gemini emits toolCall
      → Main dispatches to Selenium bridge (see Section 5)
      → Selenium executes → Main sends toolResponse back to Gemini
      → Gemini emits audio explanation → plays via 5a path

FULL DATA FLOW — PULSE LOOP (VISUAL CONTEXT):
  1. Main Pulse Loop fires every 10s (setInterval)
  2. Main captures screenshot via Electron desktopCapturer
  3. Converts to JPEG, base64 encodes
  4. Sends to Gemini WebSocket as realtimeInput.video
  5. Gemini passively observes — may proactively comment if something notable

AGENT STATE MACHINE:
  idle → listening → thinking → speaking → idle
  (Transitions broadcast via 'hud:state-update' IPC channel)

COMMAND IMPLEMENTATIONS (seleniumBridge.js):

  async function scroll_page({ pixels }) {
    await driver.executeScript(`window.scrollBy(0, ${pixels})`);
    return `Scrolled ${pixels > 0 ? 'down' : 'up'} by ${Math.abs(pixels)}px`;
  }

  async function click_element({ selector, selector_type }) {
    const by = selector_type === 'xpath' ? By.xpath(selector) : By.css(selector);
    const el = await driver.findElement(by);
    await driver.executeScript('arguments[0].click()', el);
    return `Clicked element: ${selector}`;
  }

  async function find_section({ search_text }) {
    const el = await driver.findElement(
      By.xpath(`//*[contains(text(), '${search_text.replace(/'/g, "\\'")}')]`)
    );
    await driver.executeScript(
      'arguments[0].scrollIntoView({ behavior: "smooth", block: "center" })', el
    );
    return `Found and scrolled to: "${search_text}"`;
  }

  async function fill_field({ selector, value }) {
    const el = await driver.findElement(By.css(selector));
    await el.clear();
    await el.sendKeys(value);
    return `Filled ${selector} with "${value}"`;
  }

  async function navigate({ action }) {
    const nav = driver.navigate();
    if (action === 'back')    await nav.back();
    if (action === 'forward') await nav.forward();
    if (action === 'refresh') await nav.refresh();
    return `Navigated: ${action}`;
  }

  async function open_url({ url }) {
    await driver.get(url);
    return `Opened ${url}`;
  }

  async function read_page_text({ selector }) {
    const text = await driver.executeScript(
      `return document.querySelector('${selector}')?.innerText || ''`
    );
    return text.substring(0, 4000); // Cap at 4000 chars to stay within token limits
  }

  // highlight_answer is handled by the Visual Feedback Engine (Section 6).

SECTION 6 — VISUAL FEEDBACK ENGINE
################################################################################

PURPOSE:
  When Gemini identifies an answer on the page, it calls highlight_answer().
  The engine injects a glowing highlight into the DOM, sends the toolResponse
  immediately (so Gemini can begin speaking), waits 4 seconds, then cleanly
  reverts all injected styles.

STATE MACHINE:
  IDLE → HIGHLIGHT_REQUESTED → INJECTING → HIGHLIGHTED → REVERTING → IDLE

  IDLE:                No highlight active.
  HIGHLIGHT_REQUESTED: toolCall received from Gemini.
  INJECTING:           Selenium executeScript storing original style & injecting CSS.
  HIGHLIGHTED:         Element is visually highlighted. Gemini audio is playing.
  REVERTING:           4s delay complete. Restoring original style.
                       Pulse Loop is PAUSED during REVERTING to prevent screenshot confusion.

IMPLEMENTATION (highlight_answer in seleniumBridge.js):

  const HIGHLIGHT_DURATION_MS = 4000;
  const HIGHLIGHT_STYLE = `
    outline: 3px solid #00E5FF !important;
    outline-offset: 4px !important;
    background-color: rgba(0, 229, 255, 0.12) !important;
    box-shadow: 0 0 16px rgba(0, 229, 255, 0.5) !important;
    transition: all 0.3s ease !important;
    border-radius: 4px !important;
  `;

  async function highlight_answer({ selector, explanation }) {
    // Step 1: Store original inline style
    const originalStyle = await driver.executeScript(`
      const el = document.querySelector('${selector}');
      if (!el) return null;
      return el.getAttribute('style') || '';
    `);

    if (originalStyle === null) {
      return { success: false, error: `Element not found: ${selector}` };
    }

    // Step 2: Inject highlight
    await driver.executeScript(`
      const el = document.querySelector('${selector}');
      el.style.cssText += '; ${HIGHLIGHT_STYLE.replace(/\n/g, ' ')}';
    `);

    // Step 3: Scroll into view
    await driver.executeScript(`
      document.querySelector('${selector}')
        .scrollIntoView({ behavior: 'smooth', block: 'center' });
    `);

    // Step 4: Return toolResponse IMMEDIATELY (Gemini speaks explanation concurrently)
    // The revert is scheduled as a background task — do NOT await it before responding.
    setTimeout(async () => {
      await driver.executeScript(`
        const el = document.querySelector('${selector}');
        if (el) el.setAttribute('style', '${originalStyle.replace(/'/g, "\\'")}');
      `);
      pulseLoop.resume(); // Resume Pulse Loop after revert
    }, HIGHLIGHT_DURATION_MS);

    pulseLoop.pause(); // Pause Pulse Loop during highlight

    return {
      success: true,
      explanation: explanation, // Main process must inject this as a text turn for Gemini to speak
      selector: selector
    };
  }

NOTE ON EXPLANATION DELIVERY:
  The `explanation` string in the toolResponse result is NOT automatically spoken.
  After sending toolResponse, the Main process must send a follow-up realtimeInput
  text message instructing Gemini: "Now speak this explanation aloud: <explanation>"
  OR configure the system prompt so Gemini knows to speak it automatically.
  Recommended approach: system prompt handles it (Gemini speaks after toolResponse).

SECTION 7 — THE PULSE LOOP
################################################################################

LOCATION: main.js (Electron Main Process)

CONFIGURATION:
  PULSE_INTERVAL_MS = 10_000   // 10 seconds
  PULSE_JPEG_QUALITY = 0.7     // JPEG compression quality
  PULSE_MAX_WIDTH    = 1280    // Downscale screenshot to this width max

IMPLEMENTATION:

  class PulseLoop {
    constructor(geminiWs) {
      this.geminiWs = geminiWs;
      this.paused = false;
      this.timer = null;
    }

    start() {
      this.timer = setInterval(() => this.tick(), PULSE_INTERVAL_MS);
    }

    stop() {
      clearInterval(this.timer);
    }

    pause() { this.paused = true; }
    resume() { this.paused = false; }

    async tick() {
      if (this.paused) return;
      if (this.geminiWs.readyState !== WebSocket.OPEN) return;

      try {
        const sources = await desktopCapturer.getSources({
          types: ['window', 'screen'],
          thumbnailSize: { width: PULSE_MAX_WIDTH, height: 800 }
        });

        // Target the Selenium-controlled window by title
        const target = sources.find(s => s.name.includes('Chrome')) || sources[0];
        if (!target) return;

        const jpegBase64 = target.thumbnail
          .toJPEG(Math.round(PULSE_JPEG_QUALITY * 100))
          .toString('base64');

        this.geminiWs.send(JSON.stringify({
          realtimeInput: {
            video: { data: jpegBase64, mimeType: 'image/jpeg' }
          }
        }));
      } catch (err) {
        console.error('[Pulse] Screenshot failed:', err.message);
      }
    }
  }

SECTION 8 — AUDIO PIPELINE
################################################################################

--- MIC INPUT (HUD Renderer → Gemini) ---

  File: hud/audio-capture.js

  const SAMPLE_RATE = 16000;
  const BUFFER_SIZE = 4096;

  async function startMicCapture() {
    const stream = await navigator.mediaDevices.getUserMedia({
      audio: { sampleRate: SAMPLE_RATE, channelCount: 1, echoCancellation: true }
    });

    const audioCtx = new AudioContext({ sampleRate: SAMPLE_RATE });
    const source = audioCtx.createMediaStreamSource(stream);
    const processor = audioCtx.createScriptProcessor(BUFFER_SIZE, 1, 1);

    processor.onaudioprocess = (e) => {
      const float32 = e.inputBuffer.getChannelData(0);
      const int16 = float32ToInt16(float32);
      const base64 = arrayBufferToBase64(int16.buffer);
      window.specter.sendAudioChunk(base64, 'audio/pcm;rate=16000');
    };

    source.connect(processor);
    processor.connect(audioCtx.destination);
  }

  function float32ToInt16(float32Array) {
    const int16 = new Int16Array(float32Array.length);
    for (let i = 0; i < float32Array.length; i++) {
      int16[i] = Math.max(-32768, Math.min(32767, float32Array[i] * 32768));
    }
    return int16;
  }

  function arrayBufferToBase64(buffer) {
    return Buffer.from(buffer).toString('base64');
  }

--- AUDIO OUTPUT (Gemini → Speaker) ---

  File: hud/audio-player.js

  const OUTPUT_SAMPLE_RATE = 24000;

  window.specter.onAudioOut(async ({ data, mimeType }) => {
    const pcmBuffer = Buffer.from(data, 'base64');
    const int16Array = new Int16Array(pcmBuffer.buffer);
    const float32Array = int16ToFloat32(int16Array);

    const audioCtx = new AudioContext({ sampleRate: OUTPUT_SAMPLE_RATE });
    const audioBuffer = audioCtx.createBuffer(1, float32Array.length, OUTPUT_SAMPLE_RATE);
    audioBuffer.getChannelData(0).set(float32Array);

    const source = audioCtx.createBufferSource();
    source.buffer = audioBuffer;
    source.connect(audioCtx.destination);
    source.start();
  });

  function int16ToFloat32(int16Array) {
    const float32 = new Float32Array(int16Array.length);
    for (let i = 0; i < int16Array.length; i++) {
      float32[i] = int16Array[i] / 32768.0;
    }
    return float32;
  }

SECTION 10 — STRETCH GOAL: ADAPTIVE BACKGROUND MUSIC
################################################################################

STATUS: DEFERRED. Implement only after all Sections 1–9 features are stable.

CONCEPT:
  Gemini observes the screen content via Pulse screenshots and identifies the
  activity type. Based on activity, it calls set_music_mood() to trigger
  ambient audio that matches the user's workflow.

ACTIVITY → MOOD MAPPING:
  Code editor open    → "focus"  (lo-fi beats, low volume)
  Video playing       → "mute"   (silence music to avoid conflict)
  Presentation/slides → "relax"  (soft ambient, very low volume)
  Default/browsing    → "focus"

SET_MUSIC_MOOD IMPLEMENTATION (stretch/musicBridge.js):
  const moods = {

  };

  async function set_music_mood({ mood }) {
    if (currentTrack) { currentTrack.stop(); currentTrack = null; }
    if (moods[mood]) {
      // Play via system audio or inject into HUD AudioContext
      ipcMain.emit('hud:music-play', { file: moods[mood] });
    }
    return `Music mood set to: ${mood}`;
  }
